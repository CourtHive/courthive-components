<h2>Architecture</h2>

<pre style={{ background: 'var(--chc-bg-secondary, #f5f5f5)', padding: '16px', borderRadius: '6px', fontSize: '13px', overflow: 'auto' }}>
{`src/
  components/              Component source code
    topology-builder/        Controller / Domain / Engine / UI layers
    scheduling-profile/      Controller / Domain / Engine / UI layers
    temporal-grid/           Controller / Engine / UI layers
    scoring/                 Approaches / Logic / Utils
    renderStructure/         Draw bracket rendering
    renderSchematicStructure/  Compact schematic previews
    forms/                   Declarative form system
    ...
  compositions/            Visual theme definitions
  styles/                  CSS custom properties and base styles
  stories/                 Storybook stories`}
</pre>

<h3>Key Patterns</h3>

<ul>
  <li><strong>Framework-agnostic</strong> — All components render to vanilla DOM elements. No React, Vue, or Angular dependency.</li>
  <li><strong>CSS custom properties</strong> — Theming via <code>--chc-*</code> CSS variables with light/dark mode support.</li>
  <li><strong>Factory integration</strong> — Tournament logic powered by <a href="https://courthive.github.io/competition-factory/" target="_blank">Competition Factory</a> for TODS-compliant data modeling.</li>
  <li><strong>Controller pattern</strong> — Complex features (topology builder, scheduling profile, temporal grid) use a controller layer that wires store, domain logic, and UI panels together.</li>
</ul>
